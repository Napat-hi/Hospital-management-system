## Quick Start Integration

### 1. Update API Configuration
Edit `src/api/config.js`:
```javascript
export const API_BASE_URL = 'https://your-backend-domain.com';
```

Or set environment variable in `.env`:
```
REACT_APP_API_URL=https://your-backend-domain.com
```

### 2. Replace Dummy Data
In `Adminpage.js`, replace this section:
```javascript
// Current (lines 22-32)
const dummyUsers = [...];
const listData = Array.isArray(state.listdata) && state.listdata.length > 0
  ? state.listdata
  : dummyUsers;
```

With API call:
```javascript
const [listData, setListData] = useState([]);
const [loading, setLoading] = useState(true);

useEffect(() => {
  const fetchUsers = async () => {
    try {
      const users = await userAPI.getAll();
      setListData(users);
    } catch (error) {
      console.error('Failed to fetch users:', error);
    } finally {
      setLoading(false);
    }
  };
  fetchUsers();
}, []);
```

## Backend Requirements

### Database Schema
```sql
CREATE TABLE users (
  id INT PRIMARY KEY AUTO_INCREMENT,
  first_name VARCHAR(100) NOT NULL,
  last_name VARCHAR(100) NOT NULL,
  username VARCHAR(100) UNIQUE NOT NULL,
  password VARCHAR(255) NOT NULL,  -- Store hashed password (use bcrypt)
  dob DATE NOT NULL,               -- Date of birth
  email VARCHAR(255) UNIQUE NOT NULL,
  phone VARCHAR(20),
  role ENUM('Doctor', 'Staff') NOT NULL,
  department VARCHAR(100),
  specialization VARCHAR(100),  -- For Doctors
  position VARCHAR(100),        -- For Staff
  status ENUM('active', 'disabled') DEFAULT 'active',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### Username & Password Generation
**IMPORTANT**: The frontend automatically generates username and password when creating users:

#### Username Format
- Pattern: `firstname.first3letters_of_lastname`
- Example: "John Doe" → `john.doe`
- Example: "Sarah Smith" → `sarah.smi`

#### Password Format
- Pattern: `DDMMYYYY` (from date of birth)
- Example: DOB "2004-10-17" → `17102004`
- Example: DOB "1990-05-23" → `23051990`

**Backend Security Note**: 
- The frontend sends the plain text password to the backend
- **ALWAYS hash the password before storing** (use bcrypt with salt rounds ≥ 10)
- Never store plain text passwords in the database
- Example (Node.js):
  ```javascript
  const bcrypt = require('bcrypt');
  const hashedPassword = await bcrypt.hash(password, 10);
  ```

### API Endpoints to Implement

#### GET /api/users
Returns all users (or filtered/paginated)
```json
Response: [
  {
    "id": 1,
    "first_name": "John",
    "last_name": "Doe",
    "email": "john.doe@hospital.com",
    "phone": "123-456-7890",
    "role": "Doctor",
    "department": "Cardiology",
    "specialization": "Cardiology",
    "status": "active"
  }
]
```

#### POST /api/users
Create new user (username and password are auto-generated by frontend)
```json
Request: {
  "role": "Doctor",
  "fullName": "John Doe",
  "username": "john.doe",
  "password": "17102004",
  "dob": "2004-10-17",
  "department": "Cardiology",
  "specialization": "Cardiology",
  "phone": "123-456-7890",
  "email": "john.doe@hospital.com"
}

Response: {
  "message": "User created successfully",
  "user": { 
    "id": 1,
    "first_name": "John",
    "last_name": "Doe",
    "username": "john.doe",
    "dob": "2004-10-17",
    "email": "john.doe@hospital.com",
    "role": "Doctor",
    ...
  }
}
```

**Important**: Hash the password before storing in database!

#### PUT /api/users/:id
Update existing user
```json
Request: {
  "fullName": "John Doe Updated",
  "department": "Emergency Medicine",
  ...
}

Response: {
  "message": "User updated successfully",
  "user": { ...updated user object }
}
```

#### PATCH /api/users/:id/disable
Disable user (soft delete - does not remove from database)
```json
Request: {
  "status": "disabled"
}

Response: {
  "message": "User disabled successfully",
  "user": {
    "id": 1,
    "status": "disabled",
    ...
  }
}
```
